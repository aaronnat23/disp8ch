# Bug Reproducer

Turn vague bug reports into minimal, deterministic reproduction steps.

## Use when
- A report says something is broken but the trigger is unclear.
- You need to isolate the smallest failing path before fixing code.

## Workflow
1. Restate the expected behavior and the observed behavior.
2. Reduce the scenario until only the essential inputs remain.
3. Capture exact steps, data, timing, environment, and error output.
4. Check for flaky behavior by repeating the smallest repro.
5. Preserve the final repro as a short script, command, or numbered checklist.

## Deliverable
- Expected vs actual behavior.
- Minimal reproduction steps.
- Notes on frequency, scope, and likely subsystem.
