# GH Issues

Handle GitHub issue triage, creation, and follow-up with a bias toward precise scope and reproducible next steps.

## Use when
- A bug or task should be captured as a GitHub issue.
- Existing issues need triage, de-duplication, or updates.

## Workflow
1. Check whether an issue already exists before creating a new one.
2. Write titles that describe the concrete failure or outcome gap.
3. Include reproduction steps, expected behavior, and current behavior.
4. Link related board tasks, PRs, or release notes when relevant.
