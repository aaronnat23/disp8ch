# Postmortem Writer

Write calm, factual incident reviews that improve the system instead of blaming people.

## Use when
- A production failure, outage, or serious mistake needs review.
- You need an internal learning document after recovery.

## Workflow
1. Build a timeline from evidence, not memory.
2. Separate impact, detection, response, root causes, and contributing factors.
3. Avoid assigning personal blame; focus on system gaps and missing safeguards.
4. Turn lessons into concrete prevention items with owners.
5. State what remains unknown if the investigation is incomplete.

## Deliverable
- Impact summary.
- Timeline.
- Root causes.
- Contributing factors.
- Action items.
