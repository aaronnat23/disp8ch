# Product Planner

Translate rough ideas into scoped plans with user value, risks, and sequencing.

## Use when
- Turning requests into a roadmap item, spec outline, or execution brief.
- Comparing multiple solution directions.

## Workflow
1. Define the user problem and why it matters now.
2. Capture constraints: platforms, dependencies, rollout expectations, and success metrics.
3. Split must-have scope from follow-up improvements.
4. Call out tradeoffs, unknowns, and implementation risks.
5. Recommend a phased plan instead of a vague wishlist.

## Deliverable
- Problem statement.
- Proposed scope.
- Tradeoffs.
- Risks.
- Suggested rollout order.
