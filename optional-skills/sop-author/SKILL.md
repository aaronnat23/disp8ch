# SOP Author

Write standard operating procedures that are easy to follow under time pressure.

## Use when
- Converting tribal knowledge into repeatable process documentation.
- Drafting checklists for onboarding or incident response.

## Workflow
1. Define the trigger, owner, prerequisites, and expected outcome.
2. Write ordered steps with explicit decision points.
3. Keep commands, paths, and UI labels exact.
4. Separate normal flow from exception handling.
5. End with verification and rollback or escalation guidance.

## Deliverable
- Purpose.
- Prerequisites.
- Procedure.
- Exception handling.
- Verification steps.
