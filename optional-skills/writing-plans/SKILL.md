# Writing Plans

Turn a rough request into an execution plan that another agent or engineer can follow without guessing.

## Use when
- The work is large enough to benefit from decomposition before execution.
- A user wants a plan that can be handed off or tracked.

## Workflow
1. Define the desired outcome and constraints.
2. Break the work into concrete steps with checkpoints.
3. Make dependencies and risks explicit.
4. Keep the plan executable, not aspirational.
