When to use: a slide deck or pitch presentation.

## Required sections
- Cover slide.
- Problem slide.
- Solution slide.
- Evidence slide.
- Plan / roadmap slide.
- Closing slide.

## Seed HTML instructions
One standalone HTML document with one semantic section per slide and a
`data-disp8ch-id` marker per slide so individual slides can be edited.

## Quality checklist
- One idea per slide; no overlapping elements.
- Prints cleanly (page breaks between slides).
- No invented metrics or quotes unless supplied.

## Output contract
Return only standalone HTML/CSS. Editable `data-disp8ch-id` markers required.
